noapp

Motivation

While smartphone usage is exploding, more and more service providers like e-commerce sites are pushing hard to use their mobile apps. Often these apps ask for way too many permissions, "to serve you better". We've even seen gimmicks like companies moving over to app only from PC, and then back!

This site aims to note down a NoApp Manifesto to help our smartphones.

Acceptable Capabilities for a qualifying app

  • App absolutely need to scan audio, video etc., while you could ask your customer to take a photo and then upload from gallery, this may be too confusing.

No no stuff

aka,things that websites ensure!

  • Auto start. I already have enough notifications in my life; and battery is precious. If I want to use app, I will open it!
  • Read contacts. Er, my friends aren't your friends!
  • Read SMS. Evesdropping is unethical!

Usable mobile site as an app

  • Add a nice desktop shortcut icon. If needed, prompt user to add to desktop on first use.
  • Offline data storage; it is there in HTML specs.
  • Layout; responsive design can be solved in many ways.
  • Browser compatibility; keep aside Windows Phone, and you just need to solve for WebKit engine.
  • Rich UI interaction paradigm; (a) choose simple paradigm (b) if still not satisfied, plenty of CSS and JS tricks.
